# SAMGA + brain-rw 优先级实验设计

[English](./2026-07-19-samga-brain-rw-combination-design.md)

**日期：** 2026-07-19<br>
**状态：** 建议实施<br>
**仓库：** `Human-Centered_AI_Group_Project`<br>
**实验命名空间：** `samga_brain_rw`

## 1. 目标

确定 brain-rw 中可迁移的部分——尤其是视觉低秩适配（visual low-rank
adaptation）、双时间尺度优化（two-timescale optimization）以及互补的 CLIP
相似度——能否在标准 THINGS-EEG2 零样本 EEG 到图像检索任务上改进 SAMGA。

本项目有两个不同的目标：

1. **复现目标：** 超过当前固定轮次（fixed-epoch）的本地 SAMGA
   探索性基准，即 Top-1 **89.02%**、Top-5 **98.87%**。
2. **论文数值目标：** 在本项目锁定的标准独立 200-way
   检索定义下，超过 SAMGA 论文报告的 Top-1 **91.30%**。

89.02% 到 91.30% 之间的结果代表相对于本地复现有所改进。高于 91.30%
的数值只能表述为：在本地锁定协议下数值更高；不能将其称为严格复现，也不能据此证明该方法在论文未明确说明的检查点、特征提取器和随机种子细节下超过
SAMGA。Top-5 已接近饱和，因此主要将其视为非退化指标。

## 2. 设计依据

- 经审计的 brain-rw 配方使用 BrainMLP、LAION CLIP ViT-B/32 rank-32
  视觉 LoRA，以及 TTUR（脑模型学习率 `5e-4`、视觉模型学习率
  `5e-5`）。其固定轮次、10 名受试者、5 个随机种子的结果为 Top-1
  86.66%、Top-5 98.38%。
- 一项受控的 SAMGA-on-CLIP 实验中，视觉 LoRA 得到 82.68% / 97.67%，
  冻结视觉模型时得到 76.17% / 95.79%，配对增益分别为 +6.51 和 +1.88
  个百分点。这支持测试 LoRA/TTUR，但不能证明其在 InternViT 上会获得相同增益。
- 固定轮次的本地推断版 InternViT-V2.5 基准为 89.02% / 98.87%。
  它仍属于探索性结果，因为历史上特征/检查点语义的确定受到已观察测试结果的影响。
  另外排除 91.82% 的测试集选择诊断结果，因为它直接使用正式测试集选择检查点。
- 现有 InternViT 缓存只支持冻结骨干网络的实验。在缓存特征之后训练的模块是特征空间适配器，而不是
  InternViT LoRA。真正的 LoRA 会改变内部隐藏状态，因此需要在线前向传递图像；
  如果之后冻结适配器，还需要重新生成缓存。

## 3. 不可协商的评估协议

### 3.1 划分层级与测试集重新封存

预检必须恰好找到 1,654 个非测试训练概念，否则中止并要求对协议修订进行审查。按以下方式划分：

1. 将每个概念 ID 规范化为 UTF-8。
2. 计算 `SHA256("AIAA3800-SAMGA-SPLIT-v1\n" + concept_id)`，并按十六进制摘要排序；
   若出现实际上不可能的摘要冲突，则以概念 ID 打破平局。
3. 将排名 1–200 分配给 `val-dev`，排名 201–400 分配给 `val-confirm`，
   其余 1,254 个概念分配给 `train`。
4. 对每个验证概念，将所有刺激 ID 规范化，并选择
   `SHA256("AIAA3800-SAMGA-STIM-v1\n" + split + "\n" + concept_id + "\n" +
   stimulus_id)` 摘要最小的刺激，作为其唯一 EEG 查询及匹配的图库图像。
5. 在开发期间，从梯度更新中排除 400 个验证概念的每一个刺激，并在训练前提交
   查询/图库 ID、概念 ID 及其哈希。

这样会创建四个互不重叠的决策角色：

1. **训练（Train）：** 1,254 个概念，用于开发阶段的梯度更新。
2. **开发验证（`val-dev`）：** 恰好 200 个独立 EEG 查询对 200 张图像的图库；
   用于受试者 01、05、08 及随机种子 42、43 的 pilot 选择。
3. **确认验证（`val-confirm`）：** 一个不重叠的 200 查询、200 图像任务，
   用于在相同的 10 名受试者 x 5 个随机种子网格上仅比较一次已锁定的候选方法。
4. **正式测试（Formal test）：** 原始 200 查询、200 图像任务，对所有新模型决策重新封存。

在 `val-confirm` 锁定一个最终候选方法后，使用已经锁定的轮次、训练计划和超参数，
在全部 1,654 个非测试概念的并集上，从头重新训练该候选方法及其严格匹配的对照。
该重训不使用验证、提前停止或检查点选择。这会在仅一次的正式测试之前恢复完整训练数据范围。

- 所有架构选择、超参数选择、检查点轮次、阶段晋级和停止决策均使用 `val-dev`。
- 仅在每个保留候选方法的配置冻结后，才使用 `val-confirm` 进行跨方法家族排名。
- 历史工作已经观察过正式 200-way 测试集，因此它不是完全纯净的留出集。
  尽管如此，在 Stage 0–5 期间仍将其重新封存：不得查看任何新的指标、预测或错误分析。
- 验证只选择一个最终候选方法。只有该候选方法及其配对的锁定基线进入正式的
  10 名受试者、5 个随机种子评估。
- 禁止使用测试集最佳轮次、基于测试集的提前停止以及基于测试结果的迭代决策。
- 已暴露于测试集的现有检查点只能用于工程冒烟检查或明确标注的诊断，绝不能用于最终结论。

### 3.2 检索定义

- 受试者：`01` 至 `10`。
- 最终随机种子：`42, 43, 44, 45, 46`。
- 标准独立 200-way 余弦相似度检索。
- 主要指标：50 个受试者-随机种子单元上的 Top-1 宏平均。
- 次要指标：Top-5 宏平均。
- 匈牙利算法/全局一对一分配不包含在主要实验中。即便保留，也仅作为传导式分析单独报告，
  且绝不与标准 Top-1 比较。

### 3.3 配对比较

对于每个候选方法与基线的对比单元：

- 使用相同的受试者和随机种子；
- 在架构允许的情况下，复用任务模型初始化和数据顺序哈希；
- 匹配预热、有效批量大小、优化步数、验证计划和锁定的停止轮次；
- 在运行清单中记录任何无法避免的不匹配；
- 计算每个单元的配对差值，以及受试者/随机种子双向聚类 bootstrap
  95% 置信区间。

每个最终候选方法都要报告其相对于历史 89.02% / 98.87% 基准的绝对差值，
以及其相对于严格匹配因果对照的配对差值：

- Stage 1：更强的单一分支；
- Stage 2：带有匹配参数/优化对照的冻结特征 SAMGA；
- Stage 3：在线冻结对照；
- Stage 4：共享冻结对照；
- Stage 5：最强单一分支与锁定的 Stage 1 融合中表现更好者。

### 3.4 最终成功门槛

只有在最终的 10 x 5 测试网格上同时满足以下所有条件，所选方法才会被视为稳健的本地改进：

- Top-1 绝对平均值高于历史 89.02% 基准；
- 相对于严格匹配对照的 Top-1 平均配对增益至少为 **+0.50
  个百分点**（指标采用 `[0, 1]` 尺度时为 `>= 0.005`）；
- Top-1 增益的双向聚类 bootstrap 95% CI 下界高于零；
- Top-5 平均配对增益不低于 **-0.20 个百分点**
  （在 `[0, 1]` 尺度上为 `>= -0.002`）；
- 至少 10 名受试者中的 8 名具有正的随机种子平均 Top-1 差值；
- 任一受试者的随机种子平均 Top-1 差值均不得低于 **-2.00 个百分点**
  （在 `[0, 1]` 尺度上为 `>= -0.02`）。

高于 91.30% 的结果只能描述为在本项目锁定协议下数值高于论文标题结果。
这不是精确复现，也不能证明在论文协议下具有优越性，因为准确的检查点、特征提取器和随机种子并未明确说明。
必须在报告准确率的同时报告计算成本、可训练参数数量、GPU 峰值显存、训练时间、推理编码器数量和推理延迟。

## 4. 按优先级排列的实验阶梯

下述所有阶段在锁定唯一最终候选方法之前均在验证集上进行。
某个阶段失败并不能成为打开正式测试集的理由。

### Stage 0 — 协议、来源和基线锁定

在添加方法之前：

1. 保持上游 SAMGA checkout 只读，并记录其提交。
2. 锁定推断版 InternViT V2.5 的检查点、预处理、输入分辨率、输出层、
   token 排除、池化和归一化语义。
3. 记录模型/检查点哈希和特征缓存清单。
4. 通过新的评估器复现所选冻结基线，同时在开发期间不读取正式测试指标。
5. 验证查询/图库 ID 及其顺序是显式、唯一且经过哈希的。
6. 如果后续获得作者提供的特征提取器细节，则将其视为新的预注册基线，
   而不是静默改变已锁定的基线。

退出条件：在训练/验证数据上实现确定性的基线一致性，并拥有完整的来源清单。

### Stage 1 — 仅验证集的分数级融合诊断

这是检验 InternViT-SAMGA 与 brain-rw/CLIP-LoRA 是否产生互补错误的最低成本方式。

1. 从冻结的 InternViT-SAMGA 分支和新训练的 brain-rw/CLIP-LoRA 分支输出逐查询相似度矩阵，
   并禁用正式测试集加载。仅可将历史重放用于工程诊断，不得用于候选方法选择。
2. 融合前要求查询与图库 ID/哈希完全相同。
3. 在 `val-dev` 上严格评估以下网格：
   - 逐查询 z 归一化凸融合，其中 InternViT 权重
     `alpha in {0.0, 0.1, ..., 1.0}`；
   - 温度校准的凸融合，其中每个分支的温度均取
     `{0.5, 1.0, 2.0}`，且 `alpha in {0.25, 0.50, 0.75}`；
   - 倒数排名融合，其中 `k in {10, 30, 60}`，InternViT 分支权重取
     `{0.25, 0.50, 0.75}`。
4. 只在 `val-dev` 上选择校准参数和融合权重。
5. 与更强的单一分支比较，而不仅仅与较弱分支比较。
6. 完全相同的指标先以较低计算成本打破平局，再以配置 ID 的字典序打破平局；
   不得检查单个正式测试样本。

这是集成结果，而不是单一骨干的 SAMGA 改进。它必须报告使用两个图像编码器的成本。
它还作为 Stage 5 的门控：如果 CLIP 的错误不具有互补性，则不继续训练时的双分支融合。

在受试者 01、05、08 及随机种子 42、43 上的 pilot 晋级门槛：

- 相对于最佳单一分支的 `val-dev` Top-1 增益至少为 +0.30 个百分点
  （在 `[0, 1]` 尺度上为 `>= 0.003`）；
- 6 个配对单元中至少 4 个为正；
- Top-5 平均变化不低于 -0.20 个百分点（`>= -0.002`）；
- 任一 pilot 受试者的随机种子平均 Top-1 降幅不超过 2 个百分点
  （`>= -0.02`）。

### Stage 2 — 低成本冻结特征改进与受控适配器

使用现有 InternViT 缓存，测试以下严格的单因素候选方法：

1. 显式逐层特征 LayerNorm 开启/关闭；
2. 训练集拟合的白化开启/关闭，所有统计量仅从训练集估计；
3. 共享与独立逐层图像预投影器；
4. 对锁定训练计划的最后 5 或 10 个检查点进行算术检查点平均，
   以及在相同两个窗口上进行 SWA；
5. 残差式特征空间低秩适配器。

各候选方法分别评估；本协议不允许事后组合。

对于层特征 `h_l`，初始适配器候选方法为：

```text
h'_l = h_l + gamma_l * B_l GELU(A_l LayerNorm(h_l))
```

`B_l` 采用零初始化，因此候选方法在第零步与基线完全相同，`gamma_l` 初始值为
1.0。适配器的严格网格为：

- rank in `{8, 16, 32}`；
- 适配器与任务模型学习率之比取 `{0.05, 0.10}`；
- 其他所有优化器和训练计划取值均复制自锁定基线。

必须将该适配器标注为 **缓存 InternViT 特征适配器（cached InternViT feature adapter）**，
绝不能称为 InternViT LoRA。平局依次以更高的 `val-dev` Top-5、更低的参数数量、
配置 ID 的字典序打破。

由于 SAMGA 已包含可训练的图像投影，该适配器需要与以下对照比较：

- 关闭适配器的恒等映射；
- 可训练参数数量匹配在 1% 以内的稠密瓶颈 MLP 对照；
- 参数数量相当的独立逐层预投影器；
- 所有对照使用相同的优化器、数据顺序和训练计划。

在相同 3 x 2 pilot 上的晋级门槛：

- `val-dev` Top-1 平均增益至少为 +0.50 个百分点（`>= 0.005`）；
- 6 个配对单元中至少 4 个为正；
- Top-5 平均变化不低于 -0.20 个百分点（`>= -0.002`）；
- 任一 pilot 受试者的随机种子平均 Top-1 降幅不超过 2 个百分点
  （`>= -0.02`）。

本阶段只有验证表现最佳的候选方法仍有资格继续。

### Stage 3 — 真正的 InternViT 后部 block LoRA 可行性 pilot

这是迁移 brain-rw 视觉适配的主要因果检验。为避免昂贵扩展，特意先进行 pilot。

1. 在 60 轮训练计划的第 1–45 轮，使用锁定的冻结缓存预热 SAMGA。
2. 在第 46–60 轮，在线加载 InternViT V2.5，并以 transformer block
   28 至 36（含首尾）为目标，使用锁定特征提取器的层编号约定。
3. 在每个目标 block 中，适配注意力 query/key/value/output 投影以及
   MLP `fc1`/`fc2` 线性模块。
4. Stage 0 将确切的具体模块路径写入 `lora_targets_v1.json`。
   若语义映射缺失或不明确，则中止 pilot；不得临时替换。
5. 严格评估四种配置：rank in `{4, 8}` 与视觉适配器相对任务模型的学习率比
   `{0.01, 0.05}` 交叉组合。将 LoRA alpha 设为 rank 的两倍，dropout 设为 `0.05`。
6. 保持所有 InternViT 基础参数冻结，并保留锁定的任务模型学习率和优化器。
7. 使用 BF16、梯度检查点、显存安全的 microbatch 继续训练，并记录梯度累积。
8. 使用锁定的 token 排除、patch 池化和归一化语义，输出第
   20、24、28、32、36 层的隐藏状态。

优化前：

- 固定 32 样本一致性集合上的在线冻结输出必须与缓存匹配：
  `rtol=1e-3`、`atol=1e-4`，且平均余弦相似度至少为 `0.9999`；
- 缓存与在线冻结的 Top-1/Top-5 预测和聚合指标必须完全相同；
  诊断性报告平均 top-20 集合重叠率，但不要求完整的 200 项排序逐位置匹配；
- 零 LoRA 输出必须与在线冻结输出匹配；
- 只有 LoRA 参数和预期的 SAMGA 参数可以获得梯度；
- 保存/重新加载必须保持输出和优化器状态。

每个 LoRA pilot 单元都有一个在线冻结配对对照，二者具有相同的预热、在线训练步数、
批次顺序和优化器切换。pilot 网格为受试者 01、05、08 与随机种子 42、43。
评估全部四种锁定配置。依次按 `val-dev` Top-1 平均值、Top-5、较低 rank、
较低学习率比进行选择。若未经审查修订本设计，不得引入其他 rank、block、模块、
轮次或学习率比。

晋级采用与 Stage 2 完全相同的 +0.50 个百分点门槛，并以在线冻结对照为参照。
如果没有配置通过，该方法家族停止。OOM 处理可以减小 microbatch 并增加梯度累积，
但必须保持并记录有效批量大小。

### Stage 4 — 共享的真正 LoRA、冻结并重建缓存

如果 Stage 3 通过晋级门槛，在进行 50 次独立在线运行之前，优先采用更经济的共享适配器路径：

1. 对每个随机种子，在汇集的训练受试者上训练一个受试者感知的共享 InternViT LoRA；
   梯度更新只使用训练概念。
2. 复用 Stage 3 选择的唯一 rank、目标集合、训练计划和学习率比；
   Stage 4 不引入新超参数。
3. 纳入一个使用相同训练计划的配对共享冻结对照。
4. 在创建任何正式测试缓存前锁定 LoRA 及所有选择。
5. 重新生成确定性的训练和验证 InternViT 缓存。仅当其成为唯一锁定的最终候选方法时，
   才重新生成测试缓存。
6. 下游 SAMGA 只在训练集上训练；模型选择只使用 `val-dev`。

五个最终随机种子各自训练一个独立的共享适配器，使不确定性包含适配器训练随机性。

晋级以 3 x 2 pilot 上的共享冻结对照为参照：

- `val-dev` Top-1 平均增益至少为 +0.50 个百分点（`>= 0.005`）；
- 6 个配对单元中至少 4 个为正；
- Top-5 平均变化不低于 -0.20 个百分点（`>= -0.002`）；
- 任一 pilot 受试者的随机种子平均 Top-1 降幅不超过 2 个百分点
  （`>= -0.02`）。

如果共享 LoRA 失败，但 Stage 3 联合候选方法已经通过，则只有已经锁定的 Stage 3
配置仍有资格进入确认阶段；不引入新的 rank、目标、训练计划或学习率选择。

### Stage 5 — 条件式训练时双分支后期融合

仅当 Stage 1 通过晋级门槛时才实施本阶段。不要改变 SAMGA 的受试者感知路由器，
也不要假称 CLIP 是 InternViT 的一种粒度。保留两个显式分支：

1. 分支 I 是锁定的 InternViT-SAMGA 模型。
2. 分支 C 是使用 LAION CLIP ViT-B/32 的受控 SAMGA-on-CLIP 分支。
3. 两个分支接收相同的 EEG 查询，并输出 ID 对齐且归一化的余弦相似度矩阵
   `S_I` 和 `S_C`。

使用三个可训练标量进行融合：

```text
lambda = sigmoid(a)
T_I = exp(t_I)
T_C = exp(t_C)
S_fused = lambda * S_I / T_I + (1 - lambda) * S_C / T_C
loss = CE(S_fused, y) + 0.25 * CE(S_I, y) + 0.25 * CE(S_C, y)
```

初始化 `a = t_I = t_C = 0`。在第 1–45 轮，按照各自锁定的冻结视觉训练计划，
训练两个任务分支。在第 46–60 轮，继续训练任务分支，并优化融合标量。
在冻结 CLIP 的实验臂中，CLIP 不产生梯度。在 CLIP-LoRA 实验臂中，使用 rank 32、
alpha 32、dropout `0.05`，目标模块为 `q_proj`、`k_proj`、`v_proj`、
`out_proj`、`fc1`、`fc2` 和 `visual_projection`，并使用 `0.10`
的视觉模型与任务模型学习率之比。

在 `val-dev` 上严格比较四个实验臂：仅 InternViT、仅 CLIP、带冻结 CLIP 的双分支，
以及带 CLIP-LoRA/TTUR 的双分支。

晋级以最强单一分支和锁定的 Stage 1 分数融合中表现更好者为参照：

- `val-dev` Top-1 平均增益至少为 +0.50 个百分点（`>= 0.005`）；
- 6 个配对单元中至少 4 个为正；
- Top-5 平均变化不低于 -0.20 个百分点（`>= -0.002`）；
- 任一 pilot 受试者的随机种子平均 Top-1 降幅不超过 2 个百分点
  （`>= -0.02`）。

如果未通过门槛，则该学习式分支没有保留候选方法。任何报告结果均标注为多骨干模型，
并包含两个编码器的成本。

## 5. 无测试泄漏的候选方法选择

每个阶段将 `val-dev` 指标和决策写入注册表。

完成所有适用阶段后：

1. 丢弃未通过其 `val-dev` 晋级门槛的候选方法。
2. 每个阶段最多冻结一个保留候选配置。
3. 在相同的 `val-confirm` 10 名受试者 x 5 个随机种子网格上，
   对每个保留候选方法及其严格匹配对照恰好评估一次。此评估后不得更改任何超参数。
4. 要求满足确认门槛：
   - Top-1 平均配对增益至少为 +0.50 个百分点（`>= 0.005`）；
   - 双向聚类 bootstrap 95% CI 下界高于零；
   - Top-5 平均配对变化至少为 -0.20 个百分点（`>= -0.002`）；
   - 10 名受试者中至少 8 名具有正的随机种子平均 Top-1 差值；
   - 任一受试者的随机种子平均 Top-1 差值均不得低于 -2.00 个百分点
     （`>= -0.02`）。
5. 根据 `val-confirm` Top-1 绝对平均值对确认阶段的保留候选方法排名。
   匹配对照差值是因果门槛，而不是跨方法家族排名分数。
6. 如果两个 Top-1 绝对值的差异不超过 0.10 个百分点
   （在 `[0, 1]` 尺度上为 `0.001`），则依次选择推理编码器数量较少者、
   实测 FLOPs 较低者、可训练参数较少者、Top-5 绝对值较高者，
   最后按配置 ID 的字典序选择。
7. 锁定一个配置、通用轮次策略和所有哈希。
8. 使用锁定的训练计划，在全部 1,654 个非测试概念上从头重新训练所选候选方法及其严格匹配对照。
   不再执行验证或检查点选择。
9. 创建哈希封存的最终运行清单，后续运行拒绝覆盖该清单。
10. 在正式 10 x 5 网格上恰好运行一次配对基线和所选候选方法。

如果有意将多个候选方法带入正式测试，则必须预注册这一偏离行为，
并在分析中应用适当的族级多重比较校正。默认设计只允许一个候选方法。

## 7. 验证策略

### 单元测试与集成测试

- 零初始化的特征适配器严格等于恒等映射；
- 适配器与对照的形状/参数数量正确；
- 零 LoRA 与在线冻结输出匹配；
- InternViT 基础权重保持冻结且没有梯度；
- 只有预期 LoRA 模块接收梯度；
- 检查点保存/重新加载具有确定性；
- 缓存与在线样本 ID/顺序/哈希一致；
- 训练或模型选择代码无法打开测试划分；
- 配对初始化/数据顺序哈希一致；
- 分数融合校准只使用验证集；
- 聚合器拒绝缺失/重复的 10 x 5 单元，并正确计算配对置信区间。

### 执行检查

1. 在本地运行语法、导入、单元测试和 CPU 特征适配器测试。
2. 在 `debug` 上运行单样本和单批次 GPU 冒烟测试。
3. 如果能在 30 分钟内完成，则在 `debug` 上运行一个受试者-随机种子的端到端冒烟测试。
4. 检查队列状态后，将较长的 pilot 数组提交到 `i64m1tga40u`。
5. 只扩展通过验证门槛的候选方法。

所有 `.out` 和 `.err` 文件均放在 `logs/samga_brain_rw/` 下。

## 8. 失败处理与停止规则

- **缺少分数矩阵/检查点：** 以确定性方式重新评估已锁定且重新封存测试集的检查点；
  不得为了获得有利分数而重新训练。
- **ID/顺序不匹配：** 停止融合并修复来源；绝不能仅按位置对齐行。
- **在线/缓存不匹配：** 停止真正的 LoRA 工作，直到特征提取语义一致。
- **OOM：** 减小 microbatch，启用/扩展检查点机制，并通过累积保持有效批量大小。
- **验证无增益：** 记录负面结果并停止该方法家族。
- **Stage 1 缺乏互补性：** 跳过 Stage 5。
- **特征适配器只超过不匹配对照：** 不得将增益归因于低秩。
- **共享 LoRA 失败：** 保留验证表现最佳的候选方法；不得查看测试集来选择回退方案。
- **所有方法均失败：** 发布干净的负面结果和经过验证的基线，
  而不是使用匈牙利算法或测试集选择的检查点。

## 10. 明确推迟的选项

完整 InternViT 微调、对大范围骨干区域进行 QLoRA、周期性刷新过期缓存、
替换 EEG 编码器以及蒸馏，仍可作为后续消融实验。
它们的优先级较低，因为它们会削弱归因、增加实现/计算风险，或只是近似真正的联合适配。
这些选项需要一份新审查的设计，而不能在本实验下自动启动。
